Create a dockerized n8n automated reconnaissance hub with the following requirements:

PROJECT GOAL:
Build a complete offensive security automation platform using n8n with:
- Subdomain enumeration (Subfinder, Amass)
- Network scanning (nmap with vulners/vulscan scripts, naabu)
- Web fuzzing (ffuf, katana)
- Credential testing (NetExec/nxc, hydra)
- Vulnerability scanning (nuclei)
- Exploitation tools (sqlmap, searchsploit)
- SecLists wordlists
- PostgreSQL database for storing results

STEP-BY-STEP TASKS:

1. CREATE PROJECT STRUCTURE:
   - Create directory: ~/n8n-recon-hub
   - Inside it, create: Dockerfile, docker-compose.yml, init.sql, README.md

2. BUILD DOCKERFILE:
   - Base image: n8nio/n8n:latest
   - Install system tools: nmap, nmap-scripts, whois, dig, nikto, hydra, john, netcat
   - Install Go tools: subfinder, amass, httpx, nuclei, dnsx, katana, naabu, ffuf, waybackurls, assetfinder, httprobe, gau, hakrawler
   - Install Python tools via pip: netexec, impacket, shodan, censys, requests, dnspython, sqlmap
   - Clone repositories: SecLists, sqlmap, exploitdb, vulscan (nmap script), nmap-vulners
   - Setup nmap to use vulscan and vulners scripts
   - Download common wordlists to /opt/wordlists
   - Create directories: /opt/recon-workspace, /opt/loot
   - Configure PATH for Go binaries and pipx tools
   - Set environment variable: SECLISTS_PATH=/opt/SecLists

3. CREATE DOCKER-COMPOSE.YML:
   - Service 1: n8n-recon (build from Dockerfile)
     * Port 5678:5678
     * Volumes: n8n_data, recon_workspace, loot
     * Add capabilities: NET_ADMIN, NET_RAW
     * Basic auth enabled
   - Service 2: postgres (image: postgres:15-alpine)
     * Port 5432:5432
     * Database: recon_hub
     * Mount init.sql for schema creation
   - Network: recon-net (bridge driver)

4. CREATE POSTGRESQL SCHEMA (init.sql):
   Create tables for:
   - subdomain_intel (domain, subdomain, source, ip, http_status, discovered_at, last_seen)
   - network_scans (target, scan_type, port, service, version, vulnerabilities)
   - smb_enum (target, hostname, domain, smb_version, shares, users)
   - vulnerabilities (target, vulnerability_name, severity, cvss_score, description)
   - fuzzing_results (target_url, path, status_code, content_length)
   - credentials (target, username, password, hash, service, protocol)
   Include appropriate indexes for performance

5. CREATE README.MD:
   Include:
   - Project overview
   - Build instructions (docker-compose build)
   - Deployment commands (docker-compose up -d)
   - Tool verification commands
   - Example n8n Execute Command snippets for each tool:
     * subfinder -d domain.com
     * nxc smb target -u user -p pass --shares
     * nmap -sV --script vulners target
     * nuclei -u url -severity high,critical
     * ffuf -u url/FUZZ -w /opt/SecLists/path/to/wordlist
     * sqlmap -u url --batch
   - Database connection details
   - Security warnings

6. BUILD AND VERIFY:
   - Run: docker-compose build
   - Run: docker-compose up -d
   - Verify tools are installed: docker exec n8n-recon bash -c "nxc --version && nuclei -version && subfinder -version"
   - Check PostgreSQL connection
   - Print access URL (http://localhost:5678)

IMPORTANT CONSIDERATIONS:
- Use Alpine-compatible packages (apk) where possible
- Handle pip installation with --break-system-packages flag for Alpine
- Ensure Go binaries are in PATH
- NetExec should be installed via pip3 as 'netexec' (command: nxc)
- Create proper symlinks for sqlmap and searchsploit
- Set proper permissions for workspace directories
- Include error handling in scripts

OUTPUT:
- Complete working directory structure
- All files created and ready to deploy
- Verification that build succeeds
- Summary of what was created and how to use it
